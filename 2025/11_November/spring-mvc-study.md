# 스프링 MVC 정리

스프링 MVC는 깔끔하고 느슨하게 결합된 웹 애플리케이션 아키텍처를 제공한다.  
컨트롤러, 핸들러 매핑, 뷰 리졸버 등 각각의 역할이 명확해 의존성 주입 및 자동 와이어링과 같은 스프링 핵심 기능들과 쉽게 통합될 수 있다.

---

## 🔎 자바 웹 애플리케이션 아키텍처 종류

### ✔ 모델 1 아키텍처
- JSP가 요청을 직접 처리
- JSP가 데이터 조회 + 뷰 렌더링 + 비즈니스 로직까지 처리하는 구조
- 유지보수가 어려워지고 관심 분리가 사실상 불가능

### ✔ 모델 2 아키텍처 (MVC)
- 모델: 화면 표시용 데이터 보관
- 뷰: 모델 기반 UI 렌더링
- 컨트롤러: 요청 → 모델 생성 → 뷰 선택

### ✔ 모델 2 + 프론트 컨트롤러
브라우저의 모든 요청을 단일 컨트롤러(FrontController)가 먼저 받아 공통 작업 처리 후 실제 컨트롤러에 위임하는 구조.

스프링에서는 이 역할이 **DispatcherServlet**.

---

## 🏗 스프링 MVC 처리 흐름

1. 브라우저가 URL 요청
2. DispatcherServlet이 요청 수신
3. HandlerMapping이 어떤 컨트롤러를 실행할지 결정
4. DispatcherServlet이 컨트롤러 메서드 호출
5. 컨트롤러는 ModelAndView 또는 @ResponseBody 결과 반환
6. ViewResolver가 논리적 이름 → 실제 뷰 파일 매핑
7. 뷰 렌더링
8. DispatcherServlet이 응답 반환

---

## 📌 스프링 MVC 주요 기능 요약

- 느슨한 결합의 명확한 역할 분리 구조
- 컨트롤러 메서드는 파라미터·반환 타입 정의가 매우 유연
- 도메인 객체를 그대로 Form backing object로 활용 가능
- HashMap 기반 Model 사용
- 타입 미스매치는 런타임 에러 대신 바인딩·검증 에러 처리
- MockMvc로 컨트롤러 단위 테스트 지원
