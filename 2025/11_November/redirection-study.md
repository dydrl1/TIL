## 📘 리디렉션(Redirection) 정리

### 📌 개념
리디렉션(Redirection)이란 **표준 입출력 스트림(Standard I/O stream)** 을 다른 위치로 재지정하는 것을 말한다.  
기본적으로 리눅스/유닉스 시스템에서는 다음 3가지 입출력 스트림이 존재한다.

| 스트림 | 번호 | 기본 대상 | 설명 |
|--------|------|-------------|------|
| 표준 입력(stdin) | 0 | 키보드 | 프로그램이 입력을 받는 통로 |
| 표준 출력(stdout) | 1 | 터미널 | 프로그램의 일반적인 출력 결과 |
| 표준 에러(stderr) | 2 | 터미널 | 에러 메시지 출력용 스트림 |

---

### 🧩 리디렉션의 종류

#### 1. 출력 리디렉션 (`>`, `>>`)
- `>` : 기존 파일을 **덮어씀**
- `>>` : 기존 파일에 **추가**

```bash
echo "Hello World" > result.txt
echo "New Line" >> result.txt
2. 입력 리디렉션 (<)
bash
코드 복사
cat < file.txt
3. 에러 리디렉션 (2>, 2>>)
bash
코드 복사
ls not_exist_file 2> error.log
4. 출력과 에러를 동시에 리디렉션 (&>, 2>&1)
bash
코드 복사
command &> log.txt
# 또는
command > log.txt 2>&1
⚙️ 파이프(|)와의 차이점
리디렉션은 출력의 대상(파일 등) 을 바꾸는 것이고

파이프(|) 는 출력의 흐름(다른 명령으로 전달) 을 바꾸는 것이다.

bash
코드 복사
ls > filelist.txt
ls | grep ".txt"
💡 응용 예시
bash
코드 복사
./deploy.sh > deploy.log 2>&1 &
sort < names.txt > sorted.txt
🧠 정리 요약
구문	의미
>	출력 덮어쓰기
>>	출력 추가
<	입력 리디렉션
2>	에러 리디렉션
&>	출력 + 에러 리디렉션
`	`

🏁 마무리
리디렉션은 로그 저장, 에러 추적, 배치 스크립트 자동화 등에 필수적이다.
명령의 결과를 원하는 형태로 관리할 수 있게 도와주는 강력한 도구이다.
